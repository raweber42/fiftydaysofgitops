Today, we're talking about one of ArgoCD's tiniest features, which levels up the **Developer Experience (DX)** of your platform!

Forcing a developer to navigate a separate dashboard (like Argo CD) for a quick status check creates friction and causes them to switch contexts regularly. If you want your Devs to be focused even more, give Argo's status badges a try!

A status badge is simply a dynamically image generated by Argo CD that displays the application's health and sync status (with more optional features on top).

### Visibility Everywhere (and a Direct Link!) üëÄ
By embedding the URL whereever you want, you get **instant, unauthenticated visibility** into the deployment status. Good examples of places to put the badge include:

  * **GitHub/GitLab:** In precisely the repository that the corresponding app is deploying from (see screenshot).
  * **Confluence/Wiki Pages:** In your application service catalog.
  * **Internal Dashboards:** Providing a centralized view of all applications.

Even better: the badge is typically embedded as a linked image. This means when a developer sees the badge is not green, **clicking on it forwards them directly to the Argo CD application details page**! Zero searching, just one click to the source of truth for immediate troubleshooting.


### How to Enable It üõ†Ô∏è
Activating this feature is incredibly straightforward using the Argo CD Helm chart:

    statusbadge.enabled: true

Once enabled, the badge URL follows this simple format: "${argoCdBaseUrl}/api/badge?name=${appName}".

#### Pro-Tip: Add Detail! ‚ú®

You can also include extra details to make the badge even more informative using simple query parameters:

  * `&showAppName=true`: Displays the name of the application on the badge
  * `&revision=true`: Shows the specific commit SHA being targeted

The Argo CD application details page will provide all the necessary template links (Markdown, HTML, etc.)‚Äîjust copy and paste\!
